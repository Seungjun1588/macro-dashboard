## 이 스킬의 역할

당신(메인 세션)이 오케스트레이터입니다. 아래 순서대로 직접 실행하거나 서브에이전트(Task/Agent 툴)를 호출하세요. 각 단계의 출력은 다음 단계의 프롬프트에 그대로 포함해서 넘겨야 합니다 — macro-writer와 macro-critic은 DB나 스크립트에 접근할 툴이 없으므로, 전달받은 내용이 그들이 가진 전부입니다.

## 절차

1. **데이터 확보** (직접 실행, Bash)
   - `python scripts/macro_health_check.py` 실행 → 신뢰도 결과(STALE/NEVER_COLLECTED) 확보
   - `python scripts/latest_snapshot.py` 실행 → 스냅샷 + NOTABLE 목록 확보

2. **리서치** (NOTABLE 티커가 있을 때만)
   - NOTABLE로 표시된 티커마다 `macro-researcher` 서브에이전트를 **병렬로** 호출하세요. 각 호출의 프롬프트에는 해당 티커명·이름·카테고리·최신값·변화율을 명시하세요.
   - NOTABLE 티커가 하나도 없으면 이 단계를 생략하고 3번으로 넘어가세요.

3. **작성**
   - `macro-writer` 서브에이전트를 호출하세요. 프롬프트에 1번의 신뢰도 결과 전문, 스냅샷 전문, 2번에서 받은 모든 리서치 요약을 그대로 포함하세요.
   - 반환된 초안을 다음 단계로 넘기세요.

4. **검수**
   - `macro-critic` 서브에이전트를 호출하세요. 프롬프트에 3번의 초안과, 1·2번의 원본 데이터를 함께 전달하세요.
   - "승인"이면 5번으로 진행하세요.
   - 문제가 지적되면 그 내용을 담아 `macro-writer`를 다시 호출해 수정을 요청하고, 다시 4번으로 돌아가세요. 최대 2회까지 반복하고, 그래도 해결이 안 되면 남은 문제를 사람에게 그대로 보고하고 중단하세요.

5. **저장**
   - 최종 승인된 마크다운을 `reports/<오늘 날짜, YYYY-MM-DD>.md`로 저장하세요 (Write 툴, 오케스트레이터인 당신이 직접).
   - 저장한 파일 경로를 사용자에게 알려주세요.

## 주의

- 이 파이프라인은 **사람이 직접 요청할 때만** 실행합니다 — 자동/예약 실행이 아닙니다.
- macro-writer와 macro-critic을 호출할 때 원본 데이터를 누락하지 마세요. 이들은 스스로 DB나 스크립트에 접근할 수 없습니다.
